Frequently Asked Questions

General Questions

What is Ask-Docs?

Ask-Docs is a RAG (Retrieval-Augmented Generation) based Q&A system that allows you to ask questions about your documentation and receive accurate, cited answers from a local LLM.

What file formats are supported?

Ask-Docs supports the following file formats:

  • Markdown (.md)
  • Plain text (.txt)
  • HTML (.html, .htm)
  • PDF (.pdf)
  • Excel (.xlsx, .xls)

Does it require an internet connection?

No! Ask-Docs runs entirely locally. The LLM, embedding model, and all processing happens on your machine. No data is sent to external servers.

Technical Questions

What LLM models are supported?

Ask-Docs supports any model available through Ollama. The default is llama3.2:3b which provides a good balance of quality and speed on CPU. For better quality, try llama3.2:8b or mistral.

How much memory is required?

Memory requirements depend on the model:

  • llama3.2:3b - ~4GB RAM
  • llama3.2:8b - ~8GB RAM
  • Embedding model - ~500MB RAM
  • Vector index - depends on document count

Can I use GPU acceleration?

Yes! If you have an NVIDIA GPU with CUDA support, Ollama will automatically use it. For Docker, uncomment the GPU configuration in docker-compose.yml.

How do I update the indexed documents?

Simply run the indexing endpoint again with force_reindex set to true. This will clear the existing index and rebuild it from scratch.

Troubleshooting

Why is the response slow?

On CPU, LLM inference takes time. The default model generates about 2-3 tokens per second on CPU. For faster responses, consider using a smaller model or GPU acceleration.

Why does it say "I don't have enough information"?

This happens when the system cannot find relevant information in your documents (abstention). Try:

  • Rephrasing your question
  • Making sure the topic is covered in your documents
  • Lowering the MIN_RELEVANCE_SCORE setting

The citations don't seem relevant. What can I do?

Try adjusting the chunking parameters. Smaller chunks (256-512 tokens) often work better for precise retrieval. Also ensure your documents have clear headings and structure.
